# ADR-20250517-02: ユーザーインターフェース設計

日付: 2025/05/17 00:50
ステータス: 承認

## コンテキスト
ユーザーが「キーワード」「想定読者」「画像スタイル」を入力し、大規模な記事と画像を生成するStreamlitアプリケーションのインターフェースを設計する必要があります。

## 決定
以下のUIコンポーネントとフローを実装します：

1. 入力セクション:
   - キーワード入力フィールド（必須）
   - 想定読者入力フィールド（必須）
   - 画像スタイル選択（"natural"または"vivid"）
   - 生成ボタン

2. プロセスセクション:
   - タブインターフェースで各ステップを表示
   - 進捗バーでアウトライン・記事・画像の生成進捗を表示
   - リアルタイムログ表示

3. 出力セクション:
   - アウトライン表示機能
   - 生成された記事のプレビュー（Markdownレンダリング）
   - 記事+画像のZIPダウンロードボタン

4. プロセスフロー:
   - 入力検証 → アウトライン生成 → 並行記事生成 → 並行画像生成 → 結合・プレビュー表示 → ダウンロード提供

## 影響
1. Streamlitのリアクティブな特性を活用することで、進捗状況をリアルタイムで表示可能
2. タブインターフェースにより、複数ステップのコンテンツ生成を整理して表示
3. セッション状態管理が必要になるため、Streamlitのセッション管理機能を活用
4. 長時間実行されるタスクがあるため、非同期処理をStreamlitと連携させる工夫が必要
5. ファイルのダウンロード提供のためにStreamlitのダウンロード機能を使用
